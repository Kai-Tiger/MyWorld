# 模型草优先生成玩家脚下和前方

## Context and Goal

模型草当前按玩家周围矩形扫描并逐帧生成。玩家快速移动或进入新区域时，扫描可能先处理区域边缘和身后，导致脚下/前方草慢慢加载出来。目标是在不一次性生成全地图草的前提下，让脚下和视野/移动方向前方优先生成。

## Success Criteria

- 玩家进入新区域时，脚下近处草优先出现。
- 玩家移动方向或相机朝向前方优先预热，减少前方裸地。
- 旧位置未处理的 pending 草队列不会长期抢占当前区域预算。
- 草显示半径、LOD、密度滑杆、模型草开关和风动行为保持不变。

## Planned Changes

- 将草扫描任务从矩形逐行扫描改为按 ring 从玩家中心向外扫描。
- 每个 ring 内优先处理玩家前方；`25m` 内优先按距离处理，保证脚下不空。
- 扫描预热半径扩大到 `120m`，实际显示仍由 `GRASS_LOD_HIDE_DIST` 控制。
- 地图 update 接收可选 `grassForward` context；主循环传入相机朝向，移动时地图也会用玩家位移更新方向。
- 玩家触发新扫描时清掉旧 pending 队列，只保留已经生成的缓存。

## Test/Verification Plan

- 运行 `npm run build`。
- 在草地边缘快速移动，确认脚下和前方草优先生成。
- 在 `pos=(-173.6, 20.5)` 与 `pos=(-291.8, -441.3)` 附近观察，确认不再长时间空地。
- 打开 perf 日志，确认 `map.grassQueue` 没有持续异常升高。

## Assumptions and Out of Scope

- 本次不一次性全地图生成模型草。
- 本次不改草模型、材质、风动、LOD 距离或过滤条件。
- 已生成缓存保留；仅丢弃未处理的旧 pending 队列。
