# 雪山山脚平缓化

## 背景与目标

参考图的山从草原平缓抬升、山脚宽阔铺开、整体外凸;当前 4 座雪峰山脚突兀立起像裙摆。

这与地图面积无关(世界 1600、边界 ±760,雪峰半径仅 244~306,空间充足),根因是 `src/scene/map.js` 中 `applySnowMountainHeight()` 的高度整形参数。

目标:**保留**雪峰高度、山顶岩脊与雪面,**只把平地→山的过渡带(山脚)做宽做平缓**,允许底盘适度放大。

## 成功标准

- 从草原低处平视,山脚像参考图那样平缓自然地铺开,无裙摆式陡起。
- 山顶高度与改前基本一致(峰心采样差 ~±10m),岩脊/雪线仍在。
- 沿峰心向外的高度剖面,山脚段坡度明显变缓且无台阶/悬崖突变。
- 放大后的山脚仍在 ±760 边界内,未被 `applyMountainEdgeHeight` 削出悬崖。

## 计划改动(均在 `src/scene/map.js`)

集中在 `applySnowMountainHeight()` 与 `SNOW_PEAKS` 常量,为视觉调参,初值后续配合截图/采样迭代:

1. **穹顶外凸**:各峰 `corePower` 0.86~0.94 → ~1.25~1.5(山顶不变、中段与脚部压低)。
2. **加宽裙边衰减**:`footScale` `1 - 0.8*smoothstep(0.58,1.18,d)` → `1 - 0.6*smoothstep(0.40,1.42,d)`。
3. **降低切底 + 补偿 + 加宽淡入**:`mountainLift - 100` → ~`-50`,同步各峰 `h` 下调 ~50 保持山顶;`footFade` `smoothstep(exposed,0,40)` → ~`(exposed,0,75)`。
4. **适度放大底盘**:各峰 `rx`/`rz` ×1.15~1.25,保证 `rx*1.42` 影响半径不越 `OUTDOOR_MOUNTAIN_BOUNDS`(±760),重点查最靠边的 1 号峰(x=-530)、3 号峰(x=-690)向西伸展。
5. (可选)`SNOW_RIDGE_LINES` 的 `ridgeFootScale` 同理加宽,视主峰效果再定。

不改 `heightmapTerrain.js` 全局 `sharpenPower`/`sharpenMix`。

## 验证

1. `npm run dev`,Playwright MCP + `window.__MY_GAME_DEBUG__`:冻结轨道相机、隐藏玩家,草原低处平视雪峰,改前/改后同机位截图对比。
2. `getTerrainHeight` 沿峰心(如 (-530,545))向外每 20m 采样打印剖面,确认山脚坡度变缓。
3. 边界采样确认未越界削崖;峰心采样确认山顶高度基本不变。

## 假设与范围外

- 假设:只调雪山专属整形,不动全局高度图/sharpen,不新增地形修改器。
- 范围外:不改地图尺寸/分辨率、河流/城堡/草地、雪岩贴图与法线、山顶岩脊形态。
