# 河流汇流接缝修复

## 背景与目标

河流汇流区当前显示可见的深色接缝和块状边界，多个透明河流条带重叠汇流池表面。 目标是使池表面主导汇流核心同时让普通河流条带淡出范围外之前它们堆叠在其上方。

## 成功标准

- 河流汇流区不再显示明显深色锯齿状线或矩形透明重叠阻挡。
- 河流条带仍然连接视觉上进入每个水池不带干洞。
- 现有统一水渲染保持作为一个单层合并网格和不引入 `Water.js`。
- 现有河流流动，岸边泡沫，和湖/池水类型仍然编译和渲染。

## 计划改动

- 新增一个汇流水池条带裁切遮罩派生从 `confluencePoolWeightAt()`。
- 应用该遮罩在`buildRiverStripGeometry()` 因此普通河流条带淡出范围外内部池核心。
- 把相同的水池裁切进入 `aConfluence` 因此岸线高光和水线效果是抑制在重叠过渡。
- 降低着色器 alpha/泡沫放大用于非水池汇流重叠。

## 测试与验证计划

- 运行 `npm run build`。
- 启动本地开发服务器，并确认没有出现 WebGL 着色器错误。
- 检查代表性汇流视角用于已移除的硬接缝和无明显干涸空隙。

## 假设与范围外

- 问题是主要透明水体重叠，不一个地形高度地图问题。
- 此改动不重塑河流河床，改变玩家水体物理，或重新设计河流网络。
