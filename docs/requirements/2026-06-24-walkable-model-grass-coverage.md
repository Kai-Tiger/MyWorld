# All-Land Model Grass Coverage

## Context and Goal

The current meadow grass layer is generated by broad map bounds plus height, river, slope, and clearing filters. In-game this can leave visible bare ground near the spawn area and other valid outdoor land, while recent all-land coverage can place grass on steep river banks. The goal is to cover outdoor land with 3D model grass except campfires, castle areas, actual water, and steep river-bank slopes.

## Success Criteria

- Outdoor land is covered by model grass instead of large bare patches, including the spawn area and river banks.
- Grass remains excluded from actual water surfaces.
- Grass remains excluded from steep river-bank slopes between water edge and the upper bank.
- River-bank top platforms still receive grass.
- Functional clearings remain readable: campfires and castle/castle approach areas are not filled with grass.
- Grass and world-tree generation has no artificial instance cap inside the finite outdoor map bounds.
- Meadow grass near the current spawn point `(-260, -445)` is generated first.
- Main visible grass layers are about 2x denser by area than the previous tuning.
- Existing spawn grass, riverside grass, grass variants, wind animation, LOD, and incremental build behavior remain intact.
- `npm run build` succeeds.

## Planned Changes

- Replace the meadow walkability helper in `src/scene/map.js` with a land/water helper.
- Use that helper inside `stepMeadowGrassBuild()` so grass is skipped for campfires, castle areas, actual water, and river-bank steep slopes.
- Add a river-bank steep-slope helper based on each waterway's `halfWidth` and carved bank run.
- Increase meadow, riverside, and local grass-pile density by reducing spacing by `1 / sqrt(2)`; double local grass-pile final counts.
- Remove artificial generation caps from riverside grass, meadow grass, and world trees.
- Sort meadow grass chunks by distance to the current spawn point instead of distance to world origin.
- Keep chunked `InstancedMesh` construction, grass LOD registration, and wind material setup unchanged.
- Keep grass as model instances only; do not change the terrain shader or ground material.

## Test / Verification Plan

- Run `npm run build`.
- Do not run browser, Playwright, or visual checks unless explicitly requested.

## Assumptions and Out of Scope

- "All land" means outdoor terrain within the mountain bounds that is not actual water, not a protected campfire/castle clearing, and not a steep river-bank slope.
- "No artificial cap" means no explicit max-instance cutoff in the finite map; generation is still bounded by map size and spacing.
- Grass may cover mine cave entrance areas, slopes, river banks, and other outdoor land.
- Grass should not cover the down-cut slope band from water edge to bank top.
- "2x denser" means roughly twice as many grass instances per square meter, not half spacing / 4x area density.
- Grass does not need to cover structure interiors, water, or deliberate campfire/castle clearings.
- Out of scope: new grass assets, billboard grass, terrain material changes, browser-based visual verification.
