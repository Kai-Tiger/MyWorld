# 辫状水切割重新moval

## 背景与目标

在 `pos=(-391.3, -346.2)`，多个变窄地形裁切从新区辫状河流系统是可见通过水。 它们break 水进入 hard-边缘d 阻挡和使表面看起来 sliced 由河槽山脊。

目标: 移除本地裁切伪影从这盆地因此水读作作为一个连续表面同时保留更宽辫状河流系统外部截图区域。

## 成功标准

- 水周围 `(-391.3, -346.2)` 渲染作为一个连续盆地而不是比独立d polygonal 池。
- Channel-切割山脊不再明显 slice 通过本地水面。
- 盆地仍然有一个柔和岸线和地形保持下方水内部核心。
- 现有辫状河流行为外部本地 resto比例n 盆地是不变。

## 计划改动

- 新增一个本地辫状 resto比例n 盆地围绕截图区域。
- Bl结束正常辫状场朝向一个盆地场内部该本地遮罩。
- 使用盆地场用于水遮罩，水高度，河床高度，岸线遮罩，和地形雕刻在本地核心。
- 保持河道基础的雕刻外部盆地边缘。

## 测试与验证计划

- 运行 `npm run build`。
- 检查 `(-391.3, -346.2)` 从提供的视角和验证裁切线是移除。
- 重新check 之前草裁切点 `(-448.9, -474.9)` 保持可行走。
- 检查附近辫状河流点用于无新悬空水或干涸阻挡。

## 假设与范围外

- 这移除本地可见裁切伪影，不 entire 新区辫状河流网络。
- 这不改变玩家移动阈值。
- 这不移动辫状河槽控制点。
