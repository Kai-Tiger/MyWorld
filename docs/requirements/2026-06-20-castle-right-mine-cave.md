# 城堡右侧可探索矿洞场景

## Context And Goal

在城堡入口视角右侧新增一个室外矿洞。玩家能从地面洞口沿斜坡向下进入短隧道和矿山空间，形成“地面洞口 -> 下探 -> 矿道”的场景体验。

## Success Criteria

- 矿洞位于玩家面向城堡入口时的右侧，不遮挡城堡主入口、雾门和主路线。
- 地面洞口、碎石、矿道和矿山内部空间清晰可辨。
- 洞口必须是开放缺口，不能被地面板或阻挡碰撞盖住。
- 地表不能露出大块矿洞墙体、顶板或完整地下结构。
- 洞口坡道要足够短和陡，入口通道不能被正面石块或入口碰撞挡住。
- 洞口下探方式改为楼梯；单级台阶高度必须低于玩家自动跨步上限。
- 矿洞内部需要更深，通道截面按宽高约 1:1 处理。
- 矿洞外壳只需要很薄的一层视觉皮，主要依靠碰撞边界阻挡玩家。
- 洞内不保留木支撑架。
- 洞内不输出木质构件；轨道基座和矿车使用石材或金属材质。
- 玩家可以从洞口走下斜坡进入矿洞内部，并能走回地面。
- 矿洞范围内碰撞能阻挡岩壁，斜坡和洞内地面能承载玩家。
- 城堡进入/退出逻辑、雾门、室内分区和主路线不改变。

## Planned Changes

- 新增 Blender 脚本生成 `public/models/mine_cave/mine_cave.glb`。
- 在 `src/scene/map.js` 加载矿洞 GLB，放置在城堡入口右侧的室外区域。
- 在矿洞位置局部压低 heightmap，让玩家高度逻辑允许下探。
- 洞口中心的室外地形需要被明确挖低，露出坡道和洞内空间，而不是靠模型覆盖地面假装有洞。
- 添加矿洞斜坡、洞内地面和岩壁碰撞体。
- 将洞口视觉做成环绕缺口的石边和阴影，不使用覆盖入口的整块地面板。
- 地下墙体和顶板从地面以下开始，避免在密林地表显示成白色/浅色块。
- 矿洞墙体碰撞必须基于 `MINE_CAVE` 相对坐标，避免移动入口后残留旧坐标。
- 入口处不放正面阻挡碰撞；坡道比初版更短、更陡、更宽，便于直接走入。
- 用多级楼梯替代连续坡面，视觉踏步和碰撞台阶保持一致。
- 洞底加深到约 `y=-6.4`，用 16 级约 `0.4` 高台阶连接地表和洞内空间。
- 视觉墙体/顶板做薄，删除木支撑梁；保留必要墙体碰撞。
- 清理矿洞周边森林/草卡遮挡，保证入口可见。

## Test And Verification Plan

- 运行 Blender 脚本生成矿洞 GLB。
- 运行 `npm run build`。
- 静态确认模型路径、碰撞配置和地形修改只作用于矿洞范围。
- 静态确认入口中心没有地面板或阴影块覆盖，入口坐标仍为 `x=36,z=35`。

## Assumptions And Out Of Scope

- “右侧”按玩家面向城堡入口时的右手边解释。
- 本版做同一室外场景内的可走矿洞，不做独立矿洞场景切换。
- 不新增敌人、采矿、掉落、交互 UI、传送或任务逻辑。
