# 城堡右侧可探索矿洞场景

## Context And Goal

在城堡入口视角右侧新增一个室外矿洞。地表只显示一个足够深的垂直矿洞入口和梯子，玩家靠近后通过带动画的爬梯过场下到地下矿洞，并可从洞底梯子爬回地面。

## Success Criteria

- 矿洞位于玩家面向城堡入口时的右侧，不遮挡城堡主入口、雾门和主路线。
- 地面洞口、碎石、矿道和矿山内部空间清晰可辨。
- 洞口必须是开放缺口，不能被地面板或阻挡碰撞盖住。
- 洞口位置的地形网格必须真实删面开孔，不能只把地表压低成一个浅坑。
- 地表不能露出大块矿洞墙体、顶板或完整地下结构。
- 地表只能看到一个垂直矿洞入口，不能挖开楼梯通道或洞底大厅上方地形。
- 洞口内要有可见深竖井和长梯子；玩家靠近地表洞口时出现“沿梯子下去”交互。
- 梯子视觉顶端需要微微高出洞口，地表视角能看到梯子露头。
- 洞口边缘必须用不规则岩土和碎石自然包裹，不能呈现为规则圆筒或浅色方块插在地形里。
- 梯子必须贴近竖井内壁，不能悬在洞口中心。
- 玩家不能直接走进竖井洞口卡住；进入矿洞只通过梯子交互。
- 玩家不能通过跳跃、翻滚或普通移动直接落入竖井。
- 洞底梯子附近出现“爬回地面”交互，可以回到地表入口旁安全位置。
- 梯子上下必须使用 `src/characters/player/climb.fbx` 播放爬梯动画，不能瞬间传送。
- 进入梯子后由玩家手动控制进度：`W/上方向键` 向上爬，`S/下方向键` 向下爬，松开后停在当前位置。
- 玩家抓住梯子后仍可用第三人称相机转动/缩放视角，但不能普通移动或战斗。
- 下梯子时倒放向上爬的 `climb.fbx`；上梯子时正常播放。
- 梯子过场必须沿同一个 X/Z 坐标垂直移动，并与可见梯子位置对齐，不能斜着下到矿洞。
- 爬梯期间玩家必须贴在梯子前方并面向梯子，不能漂在洞口边缘或半空。
- 地表抓住梯子的起始点必须位于洞口内侧，不能让角色悬在洞口地表上方。
- 矿洞内部需要更深，通道截面按宽高约 1:1 处理。
- 矿洞外壳只需要很薄的一层视觉皮，主要依靠碰撞边界阻挡玩家。
- 洞内不保留木支撑架。
- 洞内不输出木质构件；轨道基座和矿车使用石材或金属材质。
- 玩家可以通过梯子交互进入矿洞内部，并能通过洞底梯子回到地面。
- 竖井底部必须朝矿洞方向开口并连通地下通道，不能呈现为封闭圆井。
- 梯子底部落点必须有地下地面碰撞面，玩家爬到底后不能被默认地表高度拉回洞口顶部。
- 矿洞范围内碰撞能阻挡岩壁，洞内地面能承载玩家。
- 竖井内壁必须有独立碰撞，不能只依赖 GLB 视觉网格；底部通向矿洞的开口必须保留。
- 地下隧道和矿洞大厅的左右墙必须从洞底地面向上延伸，不能悬空导致玩家看不到墙。
- 地下隧道和矿洞大厅必须有低位顶板遮挡，玩家在洞内不能看到地表和天空。
- 竖井底部到地下隧道之间的过渡段必须封闭，只保留玩家通行口，不能从缝隙看到地表、树林或天空。
- 地下隧道走完进入矿洞大厅时，大厅入口左右两侧和上角必须封闭，不能从侧翼缺口看到地表。
- 可见墙体位置必须和碰撞边界对齐，不能出现看得到墙却能穿过，或没看到墙却被挡住。
- 玩家处于地下矿洞或爬梯下半段时，不能看到地表敌人的阴影，也不能触发地表敌人索敌、攻击音效或战斗 UI。
- 城堡进入/退出逻辑、雾门、室内分区和主路线不改变。

## Planned Changes

- 新增 Blender 脚本生成 `public/models/mine_cave/mine_cave.glb`。
- 在 `src/scene/map.js` 加载矿洞 GLB，放置在城堡入口右侧的室外区域。
- 在矿洞位置局部压低 heightmap，仅形成地表竖井入口视觉。
- 洞口中心的室外地形需要被明确挖低；挖低范围只覆盖竖井入口，不暴露地下矿洞大厅。
- 添加地表竖井阻挡、洞内地面和岩壁碰撞体。
- 将洞口视觉做成环绕缺口的石边和阴影，不使用覆盖入口的整块地面板。
- 洞口边缘使用不规则环形岩土过渡面和碎石，替代规则长方块边框。
- 地下墙体和顶板从地面以下开始，避免在密林地表显示成白色/浅色块。
- 矿洞墙体碰撞必须基于 `MINE_CAVE` 相对坐标，避免移动入口后残留旧坐标。
- 删除楼梯视觉模型、楼梯落脚面和楼梯碰撞台阶。
- 地表洞口中心加防卡阻挡；玩家靠近后进入锁控制的爬梯过场。
- 地表洞口阻挡需要覆盖跳跃高度范围，只允许通过梯子交互进入竖井。
- 洞底加深到约 `y=-12`，并保留矿洞大厅地面。
- 洞底在梯子正下方增加落脚区，并用地下通道连接到矿洞大厅。
- 洞底梯子落脚区需要独立的 `surface` 碰撞面，覆盖玩家爬梯结束位置。
- 竖井内壁底部朝地下通道方向开门洞，底部平台、通道地面和矿洞大厅视觉上连续。
- 爬梯路径使用明确的梯子锚点高度和外侧站位偏移，不再用地表安全点高度当作抓梯高度。
- 爬梯路径支持 X/Z 站位偏移；当前玩家抓梯线相对洞口中心向世界 X 负方向偏移 `2m`。
- 梯子视觉坐标、玩家爬梯线和角色朝向使用同一组 `MINE_CAVE` 参数对齐。
- 矿洞爬梯朝向使用爬梯专用修正，保证 `climb.fbx` 播放时角色正面和手脚动作朝向梯子。
- 爬梯期间禁用普通移动和战斗输入，只按上下方向控制梯子进度，到达终点后恢复玩家控制。
- 竖井模型必须开放上下端；通道顶板从竖井口后退，确保地表能看到深洞和梯子。
- 竖井梯子视觉顶端延伸到洞口上方约 `0.45m`，但玩家抓梯锚点高度不随视觉露头上移。
- 视觉墙体/顶板做薄，删除木支撑梁；保留必要墙体碰撞。
- 修正隧道和大厅墙体、顶板的垂直坐标，使墙底贴住 `bottomY`，顶板位于地下通道顶部。
- 补齐竖井底部和隧道入口之间的封闭过渡段，并缩小梯子底部地板，避免地板伸出圆井外暴露外景。
- 补齐底部通道进入大厅处的左右翼墙和前缘顶板；中间保留通道入口。
- 为竖井底部过渡段添加短侧墙碰撞，避免玩家从新增封边处穿模。
- 为大厅入口左右翼墙添加对应碰撞，避免玩家穿进大厅入口两侧漏景区。
- 为竖井内壁添加一圈分段碰撞体；底部低位碰撞跳过通道开口，上半段保持完整阻挡。
- 将矿洞大厅可走范围、墙体碰撞和可见大厅地面宽深统一到同一组 `MINE_CAVE` 参数。
- 玩家在地下矿洞范围内时，移动高度使用洞底碰撞面，不依赖把上方地形整体挖开。
- 玩家进入地下矿洞范围时，临时隐藏地表 hostile NPC 并停止其仇恨、攻击、锁定提示和音效；回到地表后恢复。
- 清理矿洞周边森林/草卡遮挡，保证入口可见。

- 运行 Blender 脚本生成矿洞 GLB。
- 运行 `npm run build`。
- 静态确认模型路径、碰撞配置和地形修改只作用于矿洞范围。
- 静态确认入口中心没有地面板或阴影块覆盖，入口坐标仍为 `x=36,z=35`。
- 静态确认 terrain `holeMasks` 对入口删面，且只对竖井入口开孔。
- 静态确认 `VIS_MINE_SHAFT_LADDER_RAIL_*` 顶端高于洞口，且 `VIS_MINE_SHAFT_LADDER_RUNG_*` 包含洞口附近的顶部横档。
- 静态确认洞口边缘使用 `VIS_MINE_SHAFT_GROUND_LIP` 和不规则碎石，不再输出 `VIS_MINE_SHAFT_RIM_*` 长方块边框。
- 静态确认没有 `VIS_MINE_STAIR_*` 和 `mine_cave_stair_surface_*` 残留。
- 静态确认 `enterMineCave` / `exitMineCave` 启动爬梯状态，不直接设置最终位置。
- 静态确认爬梯顶部和底部 X/Z 坐标一致，下降/上升路径为垂直线。
- 静态确认爬梯期间未禁用第三人称相机控制。
- 静态确认爬梯路径 X 等于洞口中心 X 减 `2m`，Z 等于可见梯子 Z 加人物站位偏移，顶部抓梯 Y 位于洞口内侧。
- 静态确认玩家爬梯时 `climb.fbx` 视觉正面朝向梯子，不背对梯子或面向洞口中心。
- 静态确认 `mine_cave_shaft_blocker` 覆盖地表跳跃高度，且不影响地下矿洞移动。
- 静态确认 `climb.fbx` 已接入玩家动画；上爬 `timeScale > 0`，下爬 `timeScale < 0`，无输入时暂停。
- 静态确认竖井底部开口、底部通道和矿洞大厅模型对象都存在，且不恢复完整封闭圆筒井壁。
- 静态确认 `mine_cave_bottom_exit_surface` 覆盖 `getMineCaveClimbBottom()` 的 X/Z 落点，高度为 `bottomY`。

## Assumptions And Out Of Scope

- “右侧”按玩家面向城堡入口时的右手边解释。
- 本版做同一室外场景内的可走矿洞，不做独立矿洞场景切换。
- 使用本地 `src/characters/player/climb.fbx`，不新增额外爬梯动画资源。
- 不新增敌人、采矿、掉落或任务逻辑。
