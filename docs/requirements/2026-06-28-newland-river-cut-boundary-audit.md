# 新区河流与G圆形切割边界审计

## 背景与目标

新区辫状河流区域可以显示水 spilling 横跨可见地面-切割条带。 当前地形代码有湿润河槽，湖，湿润冲沟，干涸冲沟，和参考地形裁切，但干涸裁切可以仍然重叠湿润河流占地范围。

目标: 明确独立湿润水 bodies 从干涸地面裁切在新区区域，和防止保持启用干涸裁切从 crossing 可见河流或湖水。

## 成功标准

- 新区湿润河流和湖保持可见仅在它们的预期河槽/湖 footprints。
- 干涸地形裁切不保持保持启用内部湿润河流/湖核心。
- 干涸裁切淡出范围外附近湿润河岸而不是 slicing 通过水面。
- 现有辫状河流连续性，湖水，湿润冲沟，和玩家水体采样保持compatible。
- `?riverAudit=1` 报告保持启用干燥-切割/潮湿-水体重叠如果它们 regress。

## 计划改动

- 新增一个共享潮湿-水体排除遮罩基于在辫状河槽，恢复盆地，静态湖，全局湿润冲沟，山体 outlet，和现有河槽网络。
- 应用该遮罩到 reference-块冲沟裁切，全局树枝状侵蚀，低地侵蚀，低地 micro 侵蚀，新区树枝状冲沟，和 east-rim 干涸河流。
- 淡出干涸裁切在湿润河岸由混合它们的地形 result 返回到 original 高度。
- 延伸现有河流审计到采样保持启用干燥-切割重叠 against 湿润水 protection zones。

## 测试与验证计划

- 运行 `npm run build`。
- 运行or 检查 `?riverAudit=1` 和确认干涸/湿润重叠报告是不存在或limited 到 inactive 淡出边缘。
- 检查报告的辫状盆地周围 `(-391.3, -346.2)`。
- 重新check `(-413.1, -326.8)`，`(-300, -410)`，`(-448.9, -474.9)`，和南侧盆地湖周围 `(-294, -710)`。

## 假设与范围外

- "新区" 表示 negative-X/negative-Z 扩展后的地形 region 控制led在`src/scene/map.js`。
- 干涸裁切可能入口路线湿润河岸，但它们的保持启用雕刻 must 被零在湿润核心和淡出附近水边缘。
- 此改动不移动河流控制点，re放置水渲染er，改变水材质，or 改变玩家移动阈值。
