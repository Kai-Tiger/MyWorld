# 新区辫状网状河谷

## Context and Goal

扩张后的 -X/-Z 新区中部有大片空地，需要切出网状河谷。目标是做成辫状河：多条浅宽河汊反复分叉、汇合，形成沙洲和连续水网；每条河汊两侧都保持约 45 度陡坡。

后续调整：仅移动新增的辫状河网到 `(-270, -450)` 附近，压缩为中型 footprint，旧主河、支流和冲沟不移动；树木使用手工沿岸林带沿新河网分布。

交汇修复：辫状河汊交互处不能出现明显白边、浅色矩形水线或内部泡沫带；交汇内部应是连续水面，只有真实岸线保留水线效果。

贴岸修复：辫状河水面不能再使用每条河汊独立矩形 strip 叠加；改用一个按河网距离场裁剪出的统一水域网格，让水面边界贴合实际河槽。

边缘平滑修复：统一水域网格不能把方格轮廓直接暴露成锯齿岸线；水面边缘需要使用连续遮罩淡出，同时保留现有动态浪花、焦散、泡沫和水面起伏。

扩张修复：新区河网需要从核心辫状河扩张成更大面积的树枝状水系，并在每个叶子河道尽头形成小湖泊；这些支流和湖泊仍然必须使用统一水面渲染。

## Success Criteria

- 新区中部出现多条互相分叉/汇合的河汊，而不是单条线性河道。
- 河网中心位于 `(-270, -450)` 附近，整体覆盖扩张到约 `x -520..-40`、`z -680..-260`。
- 每条河汊中心线低于两侧，河岸按约 45 度从河床抬回原地形。
- 河汊之间保留高于水面的沙洲/湿岸，不被整体压成平面。
- 水面、涉水判定、草木避让都使用同一套新河道采样。
- 新河网两岸有稳定的手工树木/灌木分布，避开水道核心和 45 度岸坡。
- 河汊交汇处不出现白边或矩形水面接缝。
- 新河网水面边界贴近河槽岸线，不出现大片矩形水面盖到岸坡外。
- 新河网水面边缘不出现明显方格锯齿，原有动态浪花/焦散/泡沫效果保留。
- 新增树枝状支流末端都有小湖泊，汇入口/共点处不重复生成湖泊。
- 端点小湖泊的水面、涉水判定、草木避让和地形下挖都接入现有统一水系。
- 旧出生点、城堡、矿洞、雪山和现有主河区域不被新河网改动。

## Planned Changes

- 在 `src/scene/map.js` 新增新区辫状河道定义、平滑采样、bounds 和最近河汊查询。
- 新增新区河网高度修改器，放在新区地形生成之后，按每条河汊独立切割河床与 45 度岸坡。
- 把新区河网加入统一水面渲染、统一水系采样和玩家涉水采样。
- 扩展地形材质的湿岸/岩岸着色，让新区河网有湿泥、砂砾和陡岸岩面反馈。
- 新增手工沿岸林带生成，复用现有 `forest_pack` 树、灌木、岩石以及 `loadForestGrove` 的贴地/碰撞流程。
- 给水面几何增加交汇权重，在交汇内部压低岸线泡沫、浅水绿边和边缘 alpha 伪影。
- 新辫状河水面使用统一裁剪网格，不再逐河汊生成重叠 strip。
- 给统一水面增加连续边缘遮罩属性，减少硬裁剪造成的格子锯齿，并保持主水面 shader 的动态浪花逻辑。
- 新增外围树枝状支流，并用河道叶子端点自动生成端点小湖泊。
- 将原单个新区静态湖逻辑泛化为多端点湖逻辑，统一处理湖盆、湖面、采样和植被避让。

## Test / Verification Plan

- 运行 `npm run build`。
- 使用现有 DEV 河道诊断覆盖新区河汊，检查河床、水面和坡角。
- 通过代码检查确认旧主河、支流、冲沟控制点没有被迁移。
- 通过构建确认新增水面 attribute 与合并几何一致。
- 通过代码检查确认 `createUnifiedWaterRender` 对新辫状河只加入一个统一水面 geometry。
- 通过代码检查确认动态水面 shader 的 `waterHeight`、焦散和泡沫逻辑未被移除。
- 通过代码检查确认端点湖通过 `NEWLAND_ENDPOINT_LAKES` 接入统一水面，没有逐湖创建独立水面系统。
- 保持项目要求：不做浏览器或 Playwright 验证，除非后续明确要求。

## Assumptions and Out of Scope

- 河网位于 `(-270, -450)` 附近，默认不接入旧主河；外围树枝状支流只扩张新区水系 footprint。
- 本次只做地形切割、水面和系统接入。
- 不新增桥、道路、瀑布、湖泊专用交互或手工景观摆件。
