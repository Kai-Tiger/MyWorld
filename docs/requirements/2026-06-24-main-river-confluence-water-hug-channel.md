# 主河 (-280,3) 汇流口：水面浮高、不贴河道、人物穿模

## 背景与目标

玩家在主河道 (-280,3) 站入河中时半身没入/穿模，水面浮在河床上方约 1.7m，没有贴住河道底与两岸。

headless 实测（探针 `getTerrainHeight` + 直接读 `unified_water` 网格顶点世界 Y）：
- (-280,3)：床 +1.65，水面 +3.02 → 深 1.37m
- (-274,8)：床 +1.31，水面 +3.02 → 深 1.71m（胸口）

根因：该点落在 `(-274,8)` 三河汇流水潭（`CONFLUENCE_POOLS[0]`）内。该池水面用解析河水位 `getRiverSampleAt(-274,8).waterY-0.15 ≈ +3.02`，但此处真实地形是被 `applyLargeWorldHeight` 压低的宽缓低盆（床 +0.9~+1.7），与解析水位脱钩 ~1.7m → 平坦水盘铺成浮高水板、漫过低盆、人物没胸穿模。池外主河水带=床+0.75（贴床浅水）本就正常，仅此汇流池用解析水位顶高。

目标：水深降到 ~0.7–0.9m、紧贴河床；盆缘露出成两岸；仍保持三河汇流单一连贯水面。

## 实施改动（`src/scene/map.js`，复用显式 surfaceY/bedY 机制）

`CONFLUENCE_POOLS[0]`（`{x:-274,z:8}`）由解析水面改为实测锚定：
```
{ x: -274, z: 8, rInner: 10, rOuter: 18, depth: 1.0, surfaceY: 2.0, bedY: 1.2 }
```
- `surfaceY 2.0`：贴近池外主河水带（床+0.75≈+2.0），消除 +3.02 浮高，水面降 ~1m。
- `bedY 1.2`：碗状统一盆底、消错台/沙楔（只加深 `Math.min`）。
- 盆缘（+2.0~+3.0）随水面下降经 `aSubmerge<-0.12` 着色器裁切自动露出成两岸。

## 验证（已完成）

- `npx vite build` 通过。
- 探针重测：(-280,3) 深 1.37→**0.81m**、(-274,8) 深 1.71→**0.80m**；床碳刻到 +1.2；西岸 +3.78 露出（水退）。
- headless 低视角/斜俯截图：水沉入河道、两岸抬起、不再宽大浮板；三河汇流仍单一连贯水面，无错台/沙楔/漫溢。
- 池外主河、(-160,34)/(-160,-52)/(-146,80) 其余池未受影响。

## 假设与不做项
- 仅修 (-274,8) 汇流池浮高；不重塑该处宽缓盆为深窄河谷。
- 不改水材质/透明度/焦散。
