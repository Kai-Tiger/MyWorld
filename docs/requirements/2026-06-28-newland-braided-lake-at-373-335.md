# 新区辫状盆地湖At(-373,335)

## 背景与目标

水附近 `(-373.2, -335.3)`是仍然 spl它由一个干涸条带因为区域是represented 作为一个辫状 resto比例n 盆地 plus 辫状水体-网格 masking。 它应读作作为一个湖而不是独立d 辫状河流水片。

目标: convert 这本地辫状盆地进入一个连续静态湖同时保持附近辫状河槽连接围绕湖。

## 成功标准

- 报告的区域渲染作为一个连续湖表面。
- 中央干涸条带不再独立水进入两个 pieces。
- 附近辫状河槽保持可见外部湖。
- 湖泊长 axis是窄的因此它不再上方reaches adjacent 河岸从报告的视角。
- 湖地形，水几何体，树清理，和玩法水体采样使用现有静态-湖泊路径。
- `?riverAudit=1` 不报告保持启用干燥-切割/潮湿-水体重叠用于新湖。

## 计划改动

- 新增一个新静态湖居中附近 `(-373.2, -335.3)`。
- 设置湖长半径到 `70` 同时保持其短半径，旋转，和水层级不变。
- 禁用之前本地辫状 resto比例n 盆地周围 `(-391.3, -346.2)`。
- 遮罩辫状水内部所有静态湖核心因此湖几何体 owns 这些水面。
- 让现有静态-湖泊地形雕刻和采样 handle 新湖。

## 测试与验证计划

- 运行 `npm run build`。
- 加载`?riverAudit=1` 和确认 `active_dry_cut_inside_wet_water` 保持空。
- 检查 `(-373.2, -335.3)` 从报告的视角。
- 重新check 附近辫状点周围 `(-391.3, -346.2)`，`(-413.1, -326.8)`，和 `(-300, -410)`。

## 假设与范围外

- 这区域应被一个湖，不一个辫状河流交汇。
- 现有河流控制点，水着色器，和玩家移动 rules 保持不变。
