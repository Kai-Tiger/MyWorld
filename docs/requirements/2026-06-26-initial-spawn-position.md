# 初始出生点位置

## 背景与目标

移动默认户外玩家出生点到检查的南侧盆地位置因此新 sessions 开始在请求的上方look 和相机 angle。

## 成功标准

- 一个 fresh session 开始附近 `pos=(-393.7, -505.7)` 带有 `y=66.7`。
- 玩家初始朝向读作约 `63°`。
- 初始相机视角读作约 `60°/-4°`在调试位置文本。
- 现有营火，NPC 放置，地图布局，和相机控制ler 行为保持不变。

## 计划改动

- 更新`INITIAL_PLAYER_POSITION`在`src/main.js`。
- 保持现有 conversion 位置显示的相机偏航角是存放作为 `cameraViewYaw - Math.PI`，和显示的俯仰是存放作为 negated 内部俯仰。

## 测试与验证计划

- 运行production 构建。
- 开始本地游戏和检查调试位置文本之后加载。
- 确认玩家不出生点水下，内部碰撞，or 明显 susp结束的。

## 假设与范围外

- 所请求的`朝向=63°`是玩家模型偏航角显示由调试 UI。
- 所请求的`视角=60°/-4°`是调试相机视角偏航角/俯仰显示，不内部相机控制ler 值。
- 现有浏览器 storage 和 saved 地图 layouts 是不 migrated或清理。
