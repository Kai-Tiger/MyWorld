# Grass Density Over One

## Context and Goal

The performance debug panel's `Grass Density` currently only supports reducing visible grass from existing candidates. Values above `1` should generate denser meadow grass by using a smaller sampling spacing after reload.

## Success Criteria

- `Grass Density` supports `0..2`.
- Values below `1` still reduce visible grass live.
- Values above `1` are saved and applied after reload.
- Meadow grass generation uses tighter spacing for values above `1`.
- Reset debug returns grass density to `1`.

## Planned Changes

- Read `perfDebugGrassDensity` in map code.
- Use effective meadow spacing `MEADOW_GRASS_SPACING / sqrt(density)` when density is above `1`.
- Use effective spacing for meadow candidate coordinates and scan ranges.
- Allow `setDebugGrassDensity` values up to `2`.
- Update the debug panel slider range, hover text, and reload behavior.

## Test/Verification Plan

- Run `npm run build`.
- Set `Grass Density` to `0.5` and confirm visible grass reduces without reload.
- Set `Grass Density` to `1.5` and confirm the page reloads and denser meadow candidates are generated.
- Reset debug and confirm the slider returns to `1`.

## Assumptions and Out of Scope

- This debug control only increases meadow grass generation density.
- Riverside/static grass records are not regenerated by this setting.
- The max debug multiplier is capped at `2`.
