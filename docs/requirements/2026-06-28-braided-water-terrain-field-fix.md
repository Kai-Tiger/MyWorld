# 辫状水地形Field修复

## 背景与目标

新区辫状河流区域可以产生格挡y polygon 水，硬边水边缘，重复短-距离地形山脊，和玩家裁切在草-覆盖的可行走区域。 问题是most 可见靠近远处敌人周围 `(-448.9, -474.9)` 和在辫状/湖重叠视角。

目标: 使辫状水，河床雕刻，岸线遮罩，和玩法水体采样使用一个连续 influence 场而不是独立 "best 河槽" decisions。

## 成功标准

- 辫状水不再渲染作为大型 triangular 或矩形 hard-边缘d 水片。
- 短草-覆盖的行走路径附近 `(-448.9, -474.9)` 不再包含重复隐藏 1m-缩放高度跳跃。
- 辫状河床保持下方可见水，带有无新干涸阻挡切割通过水。
- 玩家移动通过 affected 草区域不明显 sink 进入或clip 通过地形。
- 现有主河流，城堡，矿洞，和雪山体地形行为保持不变。

## 计划改动

- 新增一个连续辫状场采样器该 combines 所有附近辫状河槽进入共享 `waterMask`，`bedMask`，`shore`，`waterY`，`bedY`，深度，和 dominant-河道元数据。
- 使用该场在`applyNewlandBraidedRiverHeight()` 而不是 per-河道目标切换。
- 使用相同场在`buildNewlandBraidedWaterGeometry()` 因此水占地范围和地形雕刻匹配。
- Cull 水三角面由 per-triangle 遮罩覆盖而不是保持一个整个单元当仅一个角落有水。
- 使用相同场用于新区辫状玩法 `sampleRiver()` 和潮湿-河床草排除。

## 测试与验证计划

- 运行 `npm run build`。
- Sample 地形周围 `(-448.9, -474.9)` 和确认high-frequency neighbor 跳跃是减少。
- 检查辫状河流视角周围 `(-448.9, -474.9)`，`(-413.1, -326.8)`，`(-213.4, -386.8)`，和 `(-300, -410)`。
- 验证水保持连续和玩家可以行走通过报告的草区域不带裁切。

## 假设与范围外

- 这修复不移动辫状河槽控制点。
- 这修复不改变玩家 `AUTO_STEP`或移动物理。
- 这修复不重新设计 non-braided 水系统。
